# AGENTS.md ŌĆö Quibble (trivia repo)

Two independent apps in `apps/`; no shared root package.json or workspace tooling.

- `apps/api/` ŌĆö Go 1.23 backend
- `apps/web/` ŌĆö Next.js 15 frontend

---

## API (`apps/api/`)

All commands run from `apps/api/`. The Makefile auto-loads `.env` so env vars are inherited by all subprocesses.

```sh
make run          # go run ./cmd/server (dev server)
make build        # binary ŌåÆ ../../bin/api
make test         # go test ./... -race -count=1
make lint         # golangci-lint run ./...
make tidy         # go mod tidy
make migrate-up   # apply pending goose migrations
make migrate-down # roll back one step
make sqlc         # regenerate internal/store/ from queries/ + migrations/
```

**NEVER edit `internal/store/` by hand** ŌĆö it is fully generated by sqlc. After any SQL change in `queries/` or `migrations/`, run `make sqlc` and commit the output.

**Adding a migration:** `goose -dir migrations create <name> sql`, edit with both `-- +goose Up` and `-- +goose Down` sections, then `make sqlc`.

**IDs are UUIDv7, generated app-side** via `uuid.NewV7()`. Every `INSERT` takes an explicit `id`; the DB never generates IDs.

**Go module path:** `github.com/benbotsford/trivia`

### Required env (copy `.env.example`)

```
ADDR=:8080
DATABASE_URL=postgres://...
REDIS_ADDR=localhost:6379
REDIS_PASSWORD=
LOG_LEVEL=info
AUTH0_DOMAIN=your-tenant.us.auth0.com
AUTH0_AUDIENCE=https://api.playquibble.io
DEV_AUTH_TOKEN=   # any non-empty string locally; MUST be empty in prod
```

Redis failure is non-fatal in local dev ŌĆö CRUD routes work, WebSocket game flow does not.

## Web (`apps/web/`)

All commands run from `apps/web/`.

```sh
npm run dev    # next dev
npm run build  # next build
npm run lint   # next lint (ESLint)
npx tsc --noEmit  # typecheck (no dedicated script in package.json)
```

**No test suite configured** ŌĆö no Jest/Vitest.

### Required env

```
API_URL=http://localhost:8080   # server-side only, no NEXT_PUBLIC_ prefix
DEV_AUTH_TOKEN=                 # same value as API's DEV_AUTH_TOKEN
```

### Key conventions

- **Dev auth bypass:** `src/lib/session.ts` returns a hardcoded `DEV_SESSION` when `DEV_AUTH_TOKEN` is set ŌĆö this is the single swap point when Auth0 is wired up for real.
- **Path alias:** `@/*` ŌåÆ `src/*`
- **Route groups:** `(host)/` = authenticated host routes, `(public)/` = unauthenticated player routes. These are layout groups, not URL segments.
- **API proxy:** `next.config.ts` rewrites `/api/:path*` ŌåÆ `${API_URL}/:path*` in dev; ingress-nginx handles this in prod.
- **`src/lib/mock/`** is legacy ŌĆö being replaced by `src/lib/api/`. Prefer the real API clients.
- **TypeScript types in `src/types/index.ts` must be kept in sync manually** with `apps/api/internal/game/types.go` ŌĆö no codegen bridge exists.

## Architecture notes

- WebSocket endpoint is unauthenticated by design ŌĆö players join with only a game code and display name.
- `billing.NoopChecker{}` always grants entitlements; `subscriptions` table is intentionally unused.
- `/healthz` and `/readyz` both check Postgres + Redis connectivity.
- `seed_questions.sh` in `apps/api/` seeds dev question data.
- CI: GitHub Actions ŌĆö `ci.yml` (lint/test/typecheck on every push/PR), `release.yml` (semantic-release + Docker push on merge to `main`).
- Deployment: DigitalOcean Kubernetes, GitOps via Argo CD + Kustomize (`deploy/`).
- Docker images pushed to `ghcr.io/benbotsford/trivia-api` and `ghcr.io/benbotsford/trivia-web` on release.

## Commit format

This repo uses **Conventional Commits** ŌĆö required for semantic-release to version correctly.

```
feat: add question timer UI        # minor bump (1.x.0)
fix: correct score calculation     # patch bump (1.0.x)
feat!: redesign game state machine # major bump (x.0.0)
chore: update deps                 # no release
docs: update README                # no release
ci: update workflow                # no release
refactor: extract game helpers     # no release
```

`BREAKING CHANGE:` in the commit body also triggers a major bump. Always use the conventional prefix ŌĆö commits without one are ignored by semantic-release.
